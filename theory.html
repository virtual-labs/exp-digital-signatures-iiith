Digital Signatures Scheme

For a very brief theory of digital signature schemes and their analysis, click here

Digital signatures are cryptographic mechanisms that provide authentication, integrity, and non-repudiation for digital documents and messages. Unlike handwritten signatures, digital signatures use mathematical algorithms to ensure that a document hasn't been altered and to verify the identity of the signer.

Recipients can verify the signature using the signer's public key eee to decrypt the signature and compare it with a freshly computed hash.

RSA-Based Digital Signatures

RSA (Rivest-Shamir-Adleman) is widely used for digital signatures due to its mathematical foundation:

Key Properties:

  • Key Generation: Create a public-private key pair (e,n)(e, n)(e,n) and (d,n)(d, n)(d,n)
  • Signing: Encrypt the message hash with the private key: s=H(m)d mod ns = H(m)^d \bmod ns=H(m)dmodn
  • Verification: Decrypt the signature with the public key: H(m)=?se mod nH(m) \stackrel{?}{=} s^e \bmod nH(m)=?semodn

RSA Security Assumption: The security of RSA digital signatures relies on the difficulty of the Integer Factorization Problem: Given n=p×q, find p and q\text{Given } n = p \times q \text{, find } p \text{ and } qGiven n=p×q, find p and q

Trapdoor Function: RSA uses a trapdoor one-way function where:

  • Easy direction: Computing y=xe mod ny = x^e \bmod ny=xemodn is efficient
  • Hard direction: Computing x=yd mod nx = y^d \bmod nx=ydmodn without knowing ddd is computationally infeasible

For RSA digital signatures:

Signature Generation: s=H(m)d mod ns = H(m)^d \bmod ns=H(m)dmodn

Signature Verification: H(m)=se mod nH(m) = s^e \bmod nH(m)=semodn

Where:

  • H(m)H(m)H(m) is the hash of message mmm
  • ddd is the private key exponent
  • eee is the public key exponent
  • nnn is the modulus (n=p×qn = p \times qn=p×q for primes ppp and qqq)

RSA Key Generation Process:

  1. Choose two large prime numbers ppp and qqq
  2. Compute n=p×qn = p \times qn=p×q
  3. Compute Euler's totient function: ϕ(n)=(p−1)(q−1)\phi(n) = (p-1)(q-1)ϕ(n)=(p−1)(q−1)
  4. Choose public exponent eee such that 1<e<ϕ(n)1 < e < \phi(n)1<e<ϕ(n) and gcd⁡(e,ϕ(n))=1\gcd(e, \phi(n)) = 1gcd(e,ϕ(n))=1

Digital Signature Algorithm Steps:

Signing Process:

  1. Compute message hash: h=H(m)h = H(m)h=H(m)
  2. Generate signature: s=hd mod ns = h^d \bmod ns=hdmodn

Security Properties

Digital signatures provide three key security properties:

  1. Authentication: Verifies the identity of the message sender
  2. Integrity: Ensures the message hasn't been modified
  3. Non-repudiation: Prevents the sender from denying they signed the message
